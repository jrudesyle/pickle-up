# Contributing to PickleUp

Thank you for your interest in contributing to PickleUp! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/pickle-up.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Run tests: `npm test`
6. Commit your changes: `git commit -m "Add feature: description"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- **TypeScript**: Follow Airbnb style guide
- **Formatting**: Use Prettier (auto-format on save)
- **Linting**: ESLint must pass with no errors
- **Naming**: Use camelCase for variables, PascalCase for classes

## Commit Messages

Follow conventional commits format:
```
feat: add court filtering by amenities
fix: resolve check-in distance validation bug
docs: update API documentation
test: add unit tests for match scheduling
```

## Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Request review from maintainers
5. Address review feedback
6. Squash commits before merge

## Testing

- Write unit tests for all new functions
- Integration tests for API endpoints
- E2E tests for critical user flows
- Maintain >80% code coverage
